# User Stories — GenAI Estimation Workbench

## Intake

- As a solution lead, I want to capture opportunity context so the estimate starts from a clear client/use-case baseline.
- As a proposal owner, I want to reset only intake data so I can restart discovery inputs without losing estimation work.

## Solution Pattern

- As a solution architect, I want to select a solution pattern so downstream effort, architecture, and estimation assumptions are aligned.
- As a new user, I want to seed an example solution pattern so I can learn what good inputs look like.

## Example Estimations

- As an estimator, I want searchable example estimations so I can map my use-case description to a relevant starting point.
- As an estimator, I want coming-soon examples visible but disabled so I know what patterns are planned without accidentally seeding incomplete data.
- As an estimator, I want seeded examples for chatbot, RAG, and AI estimation so I can inspect realistic guidance before building my own estimate.

## Estimation Workspace

- As an estimator, I want estimation tabs for overview, components, WBS, resources, assumptions, risks, ROI, and delivery so I can build a complete package.
- As an estimator, I want each section to support reset, export, add, edit, and delete where data is entered so I can manage the workspace section by section.

## Components

- As an estimator, I want to capture component inventory, infra costing, and LLM sub-costing in one place so infrastructure and token costs are tied to actual solution components.

## WBS Timeline

- As a delivery lead, I want WBS rows to include start date, duration, end date, predecessor, and milestone so schedule planning is explicit.
- As a delivery lead, I want a WBS Timeline view grouped by phase and week so I can understand delivery sequencing quickly.
- As a delivery lead, I want timeline edits to update the WBS table so the chart and tabular schedule stay synchronized.

## Resources

- As a delivery manager, I want resource loading aligned to WBS phases so staffing follows the planned work.
- As a delivery manager, I want internal checks between WBS and resources so missing phase or role coverage is visible.

## Export

- As a proposal owner, I want export actions for all sections and individual sections so I can share focused outputs.
- As a proposal owner, I want export format options under the Export button so format choice does not clutter the page.
